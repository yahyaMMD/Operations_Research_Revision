1) Operations Research (OR) — What & Why

Definition. OR is the science of making better decisions using mathematical and logical methods.

Two words, two ideas

  • Operations — activities within organizations.
  • Research — apply the scientific method to find the best solutions.

What OR delivers

  • Transforms complex issues into structured, solvable models.
  • Improves decision‑making, reduces costs, increases efficiency.
  • Combines math, logic, and human factors.

2) Modeling in OR

A model is a simplified representation of reality used to understand and solve problems.

Descriptive models

Explain how a system behaves.

Mathematical models

Use equations to represent the system.

Optimization models

Find the best (maximum/minimum) outcome under constraints.

3) OR Techniques — The Toolkit

  • Linear Programming (LP)
  • Integer Programming
  • Dynamic Programming
  • Network Models
  • Nonlinear Programming

4) Phases of an OR Study

  1. Define the problem — goals, constraints, scope.
  2. Observe the system — collect/clean data; identify drivers.
  3. Build the model — translate reality into math.
  4. Solve the model — OR methods or simulation; examine trade‑offs.
  5. Validate the model — sanity‑check against history/intuition.
  6. Present results — clear recommendations to decision makers.
  7. Implement — turn insights into operating instructions.
  8. Monitor — update as objectives/data change.

5) OR — Big Picture

OR uses science to solve real‑world problems by structuring decisions, optimizing under constraints, and respecting human elements — so organizations save cost, time, and resources.

6) Linear Programming (LP) — What is it?

Definition. LP finds the best possible value (max or min) of a linear objective subject to linear constraints with continuous variables.

Model components

  • Decision variables: x₁, x₂, …, xₙ (things we control)
  • Objective: Z = c₁x₁ + c₂x₂ + ··· + cₙxₙ
  • Constraints: a₁₁x₁ + a₁₂x₂ + ··· + a₁ₙxₙ ≤ b₁, …
  • Non‑negativity: xᵢ ≥ 0

When LP applies

Relationships are linear; variables can vary continuously; coefficients are treated as known (or tested via sensitivity).

Maximize (or Minimize) Z = c₁x₁ + c₂x₂ + ··· + cₙxₙ Subject to A·x ≤ b, x ≥ 0

7) LP Assumptions

1) Proportionality

Each variable's contribution to the objective and constraints scales linearly with its level.

2) Additivity

Total effect equals the sum of individual effects; no interaction terms like x₁x₂.

3) Divisibility

Variables can take fractional values (continuous). If integers are required, use Integer Programming.

4) Certainty

All coefficients are treated as known constants. Use sensitivity analysis to test robustness.

Note. LP models are approximations — often accurate enough to guide high‑quality decisions efficiently.

8) Graphical Solution (two variables)

  1. Draw constraints as lines; shade the region satisfying all.
  2. Identify feasible region; intersections (corner points) are candidates.
  3. Plot objective as an iso‑profit/iso‑cost line Z = constant.
  4. Slide the line in the improving direction; the last touching point is optimal.
Key insight: The optimal solution of an LP occurs at a corner point of the feasible region.

9) Corner Points & Convexity

  • LP feasible regions are convex (no disconnected islands).
  • Convexity ensures no local traps: any optimum is global.
  • The Simplex method exploits this by moving from corner to corner.

10) LP Special Cases

Multiple optima: Whole edge attains the same best Z; choose by a secondary criterion.
Infeasible: No point satisfies all constraints; revisit contradictory or overly tight limits.
Unbounded: Objective can grow without limit; usually indicates a missing constraint.

11) Final Takeaway

Linear Programming turns real problems into clean math models that can be solved systematically — delivering the best outcomes while respecting constraints and resources.

12) Simplex Method — Part 1

This chapter covers how Simplex navigates the feasible region, LP standard form, basic / nonbasic variables, the minimum-ratio test, and a compact tableau view — with exam-style highlights.

Key ideas you must know

  • If an LP has an optimal solution, at least one corner (extreme) point is optimal.
  • The feasible region is convex → no local traps; a corner optimum is global.
  • Simplex moves corner to corner (i.e., BFS to better BFS) improving the objective.

LP Standard Form & conversions

  • Maximize linear objective; constraints as equalities: A·x = b; with x ≥ 0.
  • ≤ constraint → add slack (+s). ≥ constraint → add surplus (−s).
  • Unrestricted variable: replace by x = x plus − x minus, with both ≥ 0.
  • A variable or slack is zero at optimum → the corresponding constraint is binding.
Exam tip: Be able to convert any LP to standard form and identify binding vs non-binding constraints.
